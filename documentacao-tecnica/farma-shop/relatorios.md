# Relatórios

Neste menu contém todos os relatórios necessários para o estabelecimento manter um controle gerencial, de clientes e produtos e assim poder visualizar todas as informações necessárias.

### Relatório de clientes

{% tabs %}
{% tab title="Relatórios de clientes" %}
Módulo utilizado para visualizar informações de clientes referente a cadastro e compras realizadas no estabelecimento.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FjwtLlToYVqtxaYGfAbJ6%2FMenu%20inicial.png?alt=media&#x26;token=faa84a88-928e-4caa-99be-8052a382043f" alt=""><figcaption></figcaption></figure>

A seguir serão explicados todos os filtros referentes aos relatórios que podem ser utilizados.\ <mark style="color:red;">**Observação:**</mark> para pesquisa de clientes nos relatórios, podem ser utilizados nome, código ou CPF.

#### Tipos de relatório:

* **Lista de clientes:** esse relatório é utilizado para visualizar toda a lista de clientes cadastrados no sistema;\
  Mais informações em #lista-de-clientes.
* **Vendas por cliente:** esse relatório é utilizado para visualizar os registros de vendas de clientes de convênio próprio ou empresa; Mais informações em #vendas-por-cliente.\ <mark style="color:red;">**Observação:**</mark> nesse tipo de relatório, é obrigatório filtrar um cliente específico.
* **Vendas parceladas:** esse relatório é utilizado para visualizar os registros de vendas de clientes de convênio próprio ou empresa; Mais informações em #vendas-parceladas.\ <mark style="color:red;">**Observação:**</mark> nesse tipo de relatório, o usuário pode escolher por filtrar ou não o cliente/convênio.
* **Recebimentos:** esse relatório é utilizado para visualizar os registros de recebimento de convênio próprio;\
  Mais informações em #recebimentos.
* **Sem compras:** esse relatório é utilizado para visualizar a lista de clientes que não realizam compras a partir de determinada data;\
  Mais informações em #sem-compras.
* **Aniversariantes:** esse relatório é utilizado para visualizar aniversariantes de acordo com o período selecionado;\ <mark style="color:red;">**Observação:**</mark> esse relatório é exibido de acordo com a data inserida no cadastro do cliente.\
  Mais informações em #aniversariantes.
* **Uso Contínuo:** esse relatório é utilizado para visualizar os registros de venda de medicamento de uso contínuo;\ <mark style="color:red;">**Observação:**</mark> esse relatório é utilizado para vendas de farmácia popular e medicamentos que são classificados como uso contínuo no cadastro.\
  Mais informações em #uso-continuo.
* **Carta de cobrança:** esse relatório é utilizado para visualizar carta de cobrança para clientes devedores.\
  Mais informações em #carta-de-cobranca.

#### Lista de clientes

Esse relatório, pode ser utilizado de duas formas, filtrando ou não o convênio.\
Para filtrar o convênio, deve inserir o **nome** ou **código do convênio**. Com isso, serão exibidos no relatório, apenas os clientes pertencentes aquele convênio.\
Após realizar os filtros desejados, clicar em **visualizar** e o relatório será exibido.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F5GFJhahc0dIlovNY1AC1%2FLista%20de%20clientes.png?alt=media&#x26;token=36e50c53-6285-4be1-a80f-c28f898be557" alt="" width="436"><figcaption></figcaption></figure>

Com o relatório gerado, é possível visualizar as principais informações do cliente. São elas:

1. Código do cliente;
2. Nome do cliente;
3. Telefone do cliente;
4. Valor do crédito que o cliente possui para compra;
5. Código do convênio que o cliente pertence;
6. Última compra realizada;
7. Endereço;
8. Data de aniversário. \ <mark style="color:red;">**Observação:**</mark> no campo de última compra é exibida a data 01/01/1900 porque ainda não existem compras para este cliente.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FzK9eFHChxSi6LqiW69QZ%2FRelatorio.png?alt=media&#x26;token=4547a38c-9366-498f-85b9-23248420ddac" alt=""><figcaption></figcaption></figure>

#### Vendas Parceladas

Esse relatório é utilizado filtrando obrigatoriamente, o **período** e **situação**, mas ainda é possível realizar outros filtros, que serão explicados a seguir.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F9Ubq1ObKsUhHhBf33zGo%2FVendas%20parceladas.png?alt=media&#x26;token=97ab050a-43c1-4657-8051-974d2ae4df24" alt="" width="436"><figcaption></figcaption></figure>

* **Em aberto:** ao utilizar esse filtro, são exibidos apenas os registros de parcelas que ainda não foram baixadas no sistema, ou seja, que ainda não foram pagas;
* **Baixados:** ao utilizar esse filtro, são exibidos apenas os registros de parcelas que já foram baixadas no sistema, ou seja, que já foram pagas;
* **Todos:** ao utilizar esse filtro, são exibidos todos os registro, em aberto e baixados.

Após selecionar a situação desejada, é preciso selecionar um dos parâmetros:

* **Vencimento:** para esse tipo de filtro, é levado em consideração a data de vencimento das parcelas para a seleção do período;
* **Emissão:** para esse tipo de filtro, é levado em consideração a data de emissão das parcelas para a seleção do período;
* **Pagamento:** para esse tipo de filtro, é levado em consideração a data de pagamento das parcelas para a seleção do período;\ <mark style="color:red;">**Observação:**</mark> esse parâmetro apenas é disponível para a situação **baixados** e **todos**.

Além dos parâmetros anteriores, ainda existem alguns filtros, que são referente as informações que serão exibidas no relatório. Estes serão explicados a seguir.

* **Incluir bloqueados:** ao utilizar esse filtro, o relatório é gerado incluindo os clientes que estão com débito em aberto e bloqueados;\ <mark style="color:red;">**Observação:**</mark> esse filtro é utilizado apenas para a situação em aberto.
* **Um cliente por página:** ao utilizar esse filtro, o relatório é gerado incluindo cada cliente em uma página, ou seja, se forem filtrados 5 clientes, o relatório irá conter 5 páginas;
* **Incluir encargos:** ao utilizar esse filtro, o relatório é gerado com valores de encargos (juros e multa), caso tenha sido configurado no cadastro do convênio;
* **Incluir produtos:** ao utilizar esse filtro, o relatório é gerado com as informações dos produtos das vendas;
* **Apenas totais:** ao utilizar esse filtro, o relatório é gerado apenas com os valores totais das vendas.

De acordo com os filtros utilizados, o relatório poderá conter as seguintes informações:

1. Dados do cliente (código, nome e telefone);
2. Data de emissão da venda;
3. Número da venda;
4. Informação da parcela;
5. Valor da venda;
6. Valor que já foi pago;
7. Valor total de encargos em caso de atraso de pagamento, caso tenha sido configurado porcentagem de juros e multa no cadastro do convênio;
8. Saldo devedor da venda (valor da venda - valor pago);
9. Data de vencimento da parcela;
10. Código do atendente que realizou a venda;
11. Código do atendente que realizou o recebimento do convênio;
12. Informações do produto (código, descrição, valor unitário, quantidade e valor total);
13. Total do saldo devedor do cliente referente a todas as parcelas filtradas);
14. Soma dos valores de todos os clientes filtrados (valor da venda, valor pago e saldo devedor).

#### Recebimentos

Para esse relatório, podem ser utilizados alguns filtros, que são: **período**, **cliente**, **convênio**, **caixa** e **turno,** porém nenhum filtro é obrigatório, ou seja, basta clicar em **visualizar** para que o relatório seja exibido. Dessa forma, serão exibidos todos os recebimentos, independente de data e de cliente.\ <mark style="color:red;">**Observação:**</mark> o registro apenas é exibido no relatório quando é efetivado o recebimento no frente de caixa.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Frph7YkFWZY0qPSqImSdS%2FFiltro%20recebimentos.png?alt=media&#x26;token=7822d67b-4abf-4f90-8a8d-24a34fff9cf3" alt="" width="437"><figcaption></figcaption></figure>

Com o relatório gerado, é possível visualizar algumas informações. São elas:

1. Data do pagamento das parcelas;
2. Informações do cliente (nome e código);
3. Data da emissão da venda;
4. Número da venda;
5. Informações da parcela;
6. Valor total da venda;
7. Valor que já foi pago;
8. Código do atendente que realizou a venda;
9. Código do atendente que recebeu o pagamento;
10. Número do caixa que a venda foi realizada;
11. Número do turno que a venda foi realizada;
12. Tipo de pagamento do recebimento;
13. Valor total de vendas do cliente;
14. Soma do valor total de todos os clientes filtrados.

#### Itens vendidos

O relatório de itens vendidos, é utilizado para verificar as informações dos produtos e vendas dos clientes pertencentes a um determinado convênio.\
Para retirar o relatório, é necessário selecionar o **convênio**, informar o **período**, marcar o parâmetro **itens vendidos** e clicar em **imprimir**.\ <mark style="color:red;">**Observação:**</mark> para convênio empresa, é necessário selecionar o mês desejado e não informar o período.

Após utilizar os filtros, o relatório é gerado com as seguintes informações: **informações cadastrais do cliente** (código e nome), **informações da venda** (número da venda e documento), **informações do produto** (código, descrição, preço unitário, quantidade, total da venda) e **data da venda**.

#### Fechamento

O relatório de fechamento é utilizado para visualizar as vendas que constarão no fechamento daquele convênio empresa.\
Para retirar o relatório, é necessário selecionar o **convênio**, informar o **mês de fechamento**, selecionar o tipo **fechamento** e clicar em **imprimir**.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fox3DQOEOaeAPmGNuTHnZ%2FFiltro%20fechamento.png?alt=media&#x26;token=ebfeac37-4d26-42fc-8664-4cf8f1ce0f4e" alt="" width="442"><figcaption></figcaption></figure>

\
Para esse relatório, é possível utilizar ou não o parâmetro exibir relatório detalhado. Ao utilizar o parâmetro, o relatório é gerado com informações das **vendas** (número da venda, documento, informação da parcela, data da venda e valor de desconto), enquanto sem utilizar o parâmetro, o relatório é gerado apenas com os **dados do cliente** e os **valores totais da venda**.

#### Criação de pré-pedido

Este módulo é utilizado juntamente com o módulo de estoque, onde será gerado o pré-pedido. Com o pré-pedido realizado, o módulo manutenção de pedidos irá transformar o mesmo em pedido, para que o atendente possa salvá-lo para futuras cotações.\
A seguir serão explicados todos os processos que devem ser realizados no módulo de estoque.

No módulo de estoque, existem três filtros que podem ser utilizados para criar pré-pedido.

1. Estoque crítico;
2. Estoque zerado;
3. Itens vendidos.

Ao selecionar um dos filtros acima, é necessário utilizar a tecla F2, inserir a identificação do pré-pedido e o mesmo será criado. <mark style="color:red;">**Observação:**</mark> O pré-pedido é criado apenas dos produtos que necessitam de reposição, de acordo com a demanda.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FRA7NpTfvfzhZvaBh7eq4%2FCriar%20pre%20pedido.png?alt=media&#x26;token=f7df3d7a-c506-4241-b1f0-aa9f70ab23ec" alt=""><figcaption></figcaption></figure>

#### Criação de Pedido

Para criar um pedido, é necessário selecionar um pré-pedido no campo identificação do pedido, para os itens serem carregados no grid.\
Com os itens na tela, é necessário marcá-los, utilizando a tecla + e logo após clicar em gravar. Após clicar em gravar, será aberta a tela para inserir a identificação (nome) do pedido que será criado, sendo necessário apenas inserir a identificação e utilizar a tecla Enter. <mark style="color:red;">**Observações:**</mark> Todos os pré-pedidos são gravados com o prefixo PRE-. Sendo assim, ao criar um pré-pedido, não é necessário inserir o prefixo.\
Após criar um pedido, o pré-pedido é excluído automaticamente.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fc0TK3cS7SqSeki1kVpUw%2FPre%20pedido.png?alt=media&#x26;token=6ccbf7a6-d7c7-4a22-a7ab-fa51dc282452" alt=""><figcaption></figcaption></figure>

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fw3hZ04UIo1ndUXzm9qu0%2FIdentificacao%20do%20pedido.png?alt=media&#x26;token=6bb98b0e-3b7d-4981-86dc-072e07af13b0" alt=""><figcaption></figcaption></figure>

Após o pedido criado, é necessário selecionar o mesmo no campo identificação do pedido e os itens serão carregados no grid. Agora, deve ser realizada a exportação do pedido para excel. Para isso, é preciso marcar os itens, utilizando a tecla +, selecionar o tipo de saída planilha excel e clicar em enviar.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FuGgLwkxQ6RcR4KeNrx53%2FExportar%20pedido.png?alt=media&#x26;token=8db7c06b-734a-4883-97c7-b088ccab8dd8" alt=""><figcaption></figcaption></figure>

A planilha é gerada com as seguintes informações:

1. A planilha é salva com o nome dado ao pedido;
2. Informações cadastrais do produto (código de barras e descrição);
3. Último preço de custo do produto;
4. Quantidade para reposição do produto;
5. Soma da quantidade do produto de todas as lojas;
6. Campo a ser preenchido pelo cliente. Inserir o preço do fornecedor;
7. Valor total dos produtos (preço x quantidade total);
8. Campo para observações.

#### Relatório melhor compra

Com os itens do pedido ou pré-pedido carregados no grid, é possível visualizar um relatório de melhor compra dos produtos. Para isso, é necessário clicar no botão melhor compra e o relatório será aberto. \ <mark style="color:red;">**Observação:**</mark> o relatório é gerado apenas com os produtos que houveram compra nos últimos 90 dias.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Foi0wkMg4dtWOdYedeSSq%2FRelatorio%20melhor%20compra.png?alt=media&#x26;token=d6b6d3a5-6790-41eb-84af-841efae8a2e2" alt=""><figcaption></figcaption></figure>

O relatório gerado contém as seguintes informações:

1. Dados da nota fiscal (nome do fornecedor, número da nota e data de emissão da mesma);
2. Dados cadastrais do produto (código interno, descrição e fabricante);
3. Preço de custo do produto na nota;
4. Quantidade comprada do produto na nota.

#### Exclusão de pedido, pré-pedido e item

Caso seja necessário realizar a exclusão de um pré-pedido ou pedido, é preciso selecionar o mesmo e, com os itens no grid, clicar no botão excluir e o processo será realizado.\
É possível também realizar a exclusão apenas de um item. Para isso, é necessário selecionar o item, utilizar a tecla Del e o item será excluído.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FNOktGlKYw3M0xE9vP7gC%2FExclusao%20.png?alt=media&#x26;token=b0977ca6-c6e0-4d53-a6e4-a4b0921b2647" alt=""><figcaption></figcaption></figure>
