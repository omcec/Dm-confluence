# Compras

No menu compras estão as funcionalidades que o estabelecimento irá utilizar para realização de compras de produtos e processos relacionados a entrada de nota.

#### Cadastro de apresentação

Para realizar o cadastro de uma nova apresentação deve ser seguido os passos:

1. Inserir o nome da apresentação;
2. Selecionar a concentração;
3. Informar a quantidade;
4. Selecionar a unidade embalagem;
5. Selecionar a forma farmacêutica;
6. Clicar em gravar.

#### Associar produtos para apresentação

O cadastro de apresentação é realizado para associar os produtos e posteriormente será utilizado na utilização do módulo de gestor de compras.\
O processo de associação é iniciado utilizando a tecla F3. Dessa forma, será aberta a tela de apresentação.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FqKGoWDbV7CFBKsVgC4Sh%2FAssociacao%20de%20apresenta%C3%A7%C3%A3o.png?alt=media&#x26;token=4fccb70b-b71d-4d6b-9969-0faa82409ff4" alt=""><figcaption></figcaption></figure>

Para associar produtos a apresentação, deve-se primeiro realizar a pesquisa, para isso, existem alguns tipos de pesquisa, que serão citados abaixo.

* Classe;
* Código do produto;
* Descrição;
* EAN;
* Fabricante;
* Princípio ativo;
* Sub classe.

Após selecionar o filtro, informar a pesquisa e clicar em pesquisar ou utilizar a tecla F1. Ao realizar esse processo, os produtos serão exibidos no grid.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F0iDU9VvPKdsy8oMiXtRv%2FPesquisa%20de%20produtos%20apresentacao.png?alt=media&#x26;token=20027dbc-69cf-4427-a575-a21a4fd9eaac" alt="" width="560"><figcaption></figcaption></figure>

### Encomenda de produtos

{% tabs %}
{% tab title="Encomenda de produtos" %}
Este módulo é utilizado para gerenciar as demandas de encomenda de mercadoria do estabelecimento, ou seja, produtos que não possuem estoque na loja, será realizado pedido de compra para vender aquele cliente que solicitou a mercadoria.\
As informações desse módulo, serão utilizadas no módulo #gestor-de-comprasque será explicado posteriormente.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fh7chdcPST4to3mVkaC5e%2FTela%20inicial.png?alt=media&#x26;token=97fa774d-aee3-4e70-bc39-9c598d716751" alt=""><figcaption></figcaption></figure>

#### Visão geral das encomendas

Ao abrir o módulo, é possível ver uma visão geral das encomendas com as seguintes informações:

* **Pedido de compra pendente:** quantidade de registro de encomenda que ainda não foram realizados pedido de compra;
* **Pedido de compra realizado:** quantidade de registro de encomenda que já foram realizados pedido de compra;
* **Previsão para essa semana:** quantidade de registros de encomenda que estão com previsão de entrega para a semana corrente;
* **Entregue essa semana:** quantidade de registros de encomenda que já foram entregues na semana corrente.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FYvlYfkBBmgg5toikEpij%2FVisao%20geral%20das%20encomendas.png?alt=media&#x26;token=598589de-fdd5-46b4-ba26-66baec0af126" alt=""><figcaption></figcaption></figure>

#### Inclusão de encomenda

Para incluir um novo registro de encomenda, deve utilizar o botão <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F2vqqVtanVptxydnnYcWv%2FNova%20encomenda.png?alt=media&#x26;token=ffdf345f-4906-46d4-95d8-5d9429ad7b40" alt="" data-size="line"> ou a tecla F1.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FLgltbhDv0k6Y2j4AUaO5%2FIncluir%20nova%20encomenda.png?alt=media&#x26;token=f800230d-0b63-4576-8c8c-1d6aad703e12" alt=""><figcaption></figcaption></figure>

Na tela de inclusão, deve ser inserido primeiro o cliente da encomenda, que pode ser pesquisado por nome ou código interno através do campo cliente ID.\
Caso o cliente não esteja cadastrado, pode ser acessado o cadastro de clientes através da tecla **\[F3]**.\
O parâmetro **\[F2] - Encomenda com vários itens para este cliente**, pode ser marcado quando serão lançados vários produtos para o mesmo cliente.\
Após os dados do cliente, deve ser pesquisado o produto para encomenda. A pesquisa pode ser realizada por nome, código de barras ou código interno.\
Ao pesquisar o produto, as informações cadastrais do produto (código, descrição, fabricante, classe e estoque atual) são preenchidas, sendo necessário inserir a quantidade para encomenda, definir uma data de previsão de entrega da encomenda e se necessário, inserir uma observação.\
Com as informações preenchidas, clicar no botão <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FHEoiMFGkloQVfbLstfPn%2FBotao%20incluir%20item.png?alt=media&#x26;token=1727153d-52c8-4a86-a1e9-49bba7faa4c5" alt="" data-size="line">ou utilizar a tecla **\[F1].**

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FobpfU1z6Knv7AOVGyu8M%2FInclusao%20de%20encomenda.png?alt=media&#x26;token=105a1f49-3141-4172-a992-c2b8109bfee2" alt=""><figcaption></figcaption></figure>

<mark style="color:red;">**Observações:**</mark>

* Durante a inclusão de itens, caso o parâmetro **\[F2] -Encomenda com vários itens para este cliente** não estiver habilitado, ao finalizar o lançamento do produto, será exibida uma mensagem perguntando ao usuário se ele deseja realizar um novo lançamento para este mesmo cliente. Enquanto, com o parâmetro habilitado, a menagem não é exibida, quando finalizar o lançamento de um produto, a tela não é fechada e o usuário pode realizar outros lançamentos.
* Caso a data de previsão de entrega seja definida para o dia corrente, é exibida uma mensagem de alerta, porém, o lançamento é realizado normalmente caso o usuário confirme.
* Se a quantidade para encomenda está disponível em estoque, é exibida uma mensagem de alerta, porém, o lançamento é realizado normalmente caso o usuário confirme.

#### Pesquisa

Após a inclusão de uma encomenda, é possível realizar filtros para pesquisa. * Produto ID;
* Últimas 10 encomendas;

Após realizar a pesquisa desejada e clicar no botão <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FKb5DnQ7tSw9N2YTxntGr%2FBotao%20localizar.png?alt=media&#x26;token=23d798f2-c250-4066-8e18-fc61c131da55" alt="" data-size="line"> ou utilizar a tecla **\[F5]**, os registros serão exibidos no grid com as informações do cliente e do produto.

#### Detalhes da encomenda

Ao selecionar um registro de encomenda, é possível visualizar os detalhes da mesma. As informações serão citadas e explicadas abaixo.

* **Usuário do cadastro:** código e nome do usuário que realizou a inclusão da encomenda;
* **Previsão de entrega:** data de previsão de entrega para a encomenda;
* **Data de cadastro:** data e hora que a encomenda foi incluída no sistema;
* **Usuário da entrega:** código e nome do usuário que registrou a entrega da encomenda;
* **Entregue:** este campo pode conter as informações SIM (quando a encomenda já foi entregue) ou NÃO (quando a encomenda ainda não foi entregue);
* **Data da entrega:** data e hora que a encomenda foi entregue;
* **Observação:** informação de observação que foi incluída no momento do cadastro da encomenda.\ <mark style="color:red;">**Observação:**</mark> é possível alterar a informação do campo observação, utilizando a tecla **\[F6].**

### Gestor de compras

{% tabs %}
{% tab title="Gestor de compras" %}
Esse módulo é utilizado para gerenciar as compras do estabelecimento, ou seja, será possível gerir e encontrar o melhor preço para compra dos produtos nas distribuidoras.

Na utilização do módulo, é possível utilizar a integração com a empresa SmartPed, que é responsável por realizar as cotações dos produtos nas distribuidoras. Essa integração é um serviço pago, onde será gerado um contrato com o estabelecimento e ele poderá utilizar a ferramenta por completo. Porém, o estabelecimento que não adquirir a integração, também poderá utilizar o módulo, porém, de uma forma mais simples, que será explicado posteriormente.

### Conceder acesso ao módulo

Inicialmente, por padrão, nenhum usuário possui acesso ao módulo. Para habilitar o acesso, é preciso acessar o cadastro do usuário e conceder acesso ao módulo e as funções do mesmo.

#### Tipos de relatório

* **Caderno de faltas:** para esse tipo de relatório, são exibidos todos os registros que foram lançados no módulo de caderno de faltas;
* **Encomenda de clientes:** para esse tipo de relatório, são exibidos todos os registros que foram lançados no módulo de encomenda de produtos;
* **Estoque menor que a demanda:** para esse tipo de relatório, são exibidos todos os produtos que o estoque atual está menor que a demanda;
* **Itens vendidos:** para esse tipo de relatório, são exibidos todos os itens vendidos de acordo com o período que será informado pelo usuário;
* **Itens vendidos - Curva ABC:** para esse tipo de relatório, são exibidos todos os itens vendidos de acordo com a curva ABC e o período informado pelo usuário;
* **Reposição da quantidade vendida:** para esse tipo de relatório, são exibidos todos os itens e sua respectiva quantidade vendida no período informado pelo usuário;
* **Todos os produtos:** para esse tipo de relatório, são exibidos todos os produtos. Nesse caso, o usuário poderá utilizar outros filtros para melhorar a pesquisa.

#### Filtros

* **Reposição de estoque para X dias:** nesse campo, o usuário deve inserir a quantidade de dias ele deseja para reposição dos itens;\ <mark style="color:red;">**Observações:**</mark>\
  Por padrão, o sistema traz a quantidade de 7 dias;\
  Esse campo só é possível alterar para os tipos de relatório: estoque menor que demanda, itens vendidos, itens vendidos - curva ABC, todos os produtos.
* **Remover itens sem sugestão de compra:** esse parâmetro é utilizado para que os itens que não possuem sugestão de compra, ou seja, produtos que não precisam de reposição, não sejam exibidos no relatório;\ <mark style="color:red;">**Observação:**</mark> só é possível utilizar este parâmetro para os tipos de relatório: itens vendidos, itens vendidos - curva ABC e todos os produtos.
* **Pedir sempre 1 (um) para estoque zero:** esse parâmetro é utilizado para que itens que possuam sugestão de compra igual a 0, seja inserida 1 unidade no produto para reposição;\ <mark style="color:red;">**Observação:**</mark> só é possível utilizar este parâmetro para os tipos de relatório: estoque menor que demanda, itens vendidos, itens vendidos - curva ABC e todos os produtos.
* **Ordenamento dos itens:** esse parâmetro é utilizado para selecionar a ordenação em que os itens serão exibidos no relatório. Esta ordenação pode ser por: descrição, classe, código do produto, estoque e fabricante; \ <mark style="color:red;">**Observação:**</mark> esse parâmetro só é utilizado para os tipos de relatório: caderno de faltas, encomenda de clientes, estoque menor que demanda e todos os produtos.
* **Classe/subclasse:** esse parâmetro é utilizado para selecionar a(s) classe(s)/subclasse(s). Dessa forma, serão exibidos apenas os produtos que pertencem aquela classe/subclasse;
* **Fabricante:** esse parâmetro é utilizado para selecionar o(s) fabricante(s). Dessa forma, serão exibidos apenas os produtos que pertencem aquele fabricante;
* **Selecione os itens que a curva do produto seja:** esse parâmetro é utilizado para selecionar a curva (A,B ou C). Dessa forma, serão exibidos apenas os produtos que pertencem aquela curva;\ <mark style="color:red;">**Observação:**</mark> esse parâmetro só é utilizado para os tipos de relatório: estoque menor que demanda, itens vendidos, itens vendidos - curva ABC, reposição da quantidade vendida e todos os produtos.
* **Selecione os itens que possuam as apresentações:** esse parâmetro é utilizado para selecionar a apresentação. Dessa forma, serão exibidos apenas os produtos que são associados aquela apresentação;
* **Selecione os itens que a situação do produto seja:** esse parâmetro é utilizado para selecionar a situação do produto (ativo ou inativo). De acordo com o filtro, os produtos serão exibidos;\ <mark style="color:red;">**Observação:**</mark> esse parâmetro só é utilizado para os tipos de relatório: estoque menor que a demanda, itens vendidos, itens vendidos 0 curva ABC, reposição da quantidade vendida, todos os produtos.
* **Selecione os itens com a situação do estoque:** esse parâmetro é utilizado para selecionar a situação do estoque dos itens, que são as seguintes: itens que o estoque está igual a 0, itens que o estoque está maior que 0, itens que o estoque está menor que a demanda;
* **Data inicial e data final:** parâmetro para inserir o período para filtro;\ <mark style="color:red;">**Observação:**</mark> esse filtro é utilizado apenas para os tipos de relatório: itens vendidos, itens vendidos - curva ABC, reposição da quantidade vendida.
* **Remover itens das vendas de PBMs:** esse parâmetro é utilizado para remover itens das vendas de PBM, ou seja, os produtos que foram vendidos por PBM, não serão exibidos. Ainda é possível selecionar o(s) PBM's para remoção;\ <mark style="color:red;">**Observação:**</mark> esse filtro é utilizado apenas para os tipos de relatório: itens vendidos e reposição da quantidade vendida.
* **Remover os itens que estejam nas cotações de compra:** esse parâmetro é utilizado para selecionar as cotações que já foram criadas. Dessa forma, os itens que constam nas cotações selecionadas não serão exibidos;
* **Remover os itens que estejam nos pedidos de compra:** esse parâmetro é utilizado para selecionar os pedidos de compra já realizados. Dessa forma, os itens que constam nos pedidos de compra selecionados não serão exibidos;
* **Exibir itens que estejam em processo de cotação de compra:** esse parâmetro é utilizado para exibir os itens que já estão em algum registro de cotação de compra criado anteriormente.\ <mark style="color:red;">**Observação:**</mark> esse filtro é utilizado apenas para os tipos de relatório: caderno de faltas e encomendas de clientes.

#### Tipos de pesquisa

Neste menu existem alguns tipos de pesquisa, para que sejam realizados filtros para melhor visualização dos registros de cotação.

* **Cancelada:** ao utilizar esse tipo de pesquisa, são exibidos no grid apenas os registros de cotação cancelada;
* **Consulta de retorno pendente:** nesse tipo de pesquisa, são exibidos no grid apenas os registros de cotação que estão pendente de retorno da distribuidora;
* **Data de cadastro da cotação:** nesse tipo de pesquisa, é necessário inserir um período e de acordo com ele, serão exibidos no grid as cotações de compra que foram cadastradas no período informado;
* **Envio de cotação pendente:** para esse tipo de pesquisa, são exibidos no grid apenas cotações de compra que estão pendente de envio para a distribuidora;
* **Identificação:** nesse tipo de pesquisa, é necessário inserir uma identificação de cotação para que seja pesquisada nos registros de cotações já criadas. Dessa forma, serão exibidos apenas os registros de cotação de compra que possuem aquela identificação;\ <mark style="color:red;">**Observação:**</mark> caso possua mais de um registro de cotação que contém a mesma palavra que está sendo pesquisada, todos os registros serão exibidos.
* **Nome do usuário - cadastro:** para esse tipo de pesquisa, é necessário inserir um nome de usuário para pesquisa. Nesse caso, serão exibidos no grid todas as cotações cadastrados por aquele usuário;
* **Últimas 15 cotações:** nesse tipo de pesquisa, são exibidos no grid, as últimas 15 cotações que foram criadas;
* **Usuário ID - cadastro:** para esse tipo de pesquisa, é necessário inserir um código de usuário para pesquisa. Nesse caso, serão exibidos no grid todos os registros de cotação cadastrados por aquele usuário;

Além dos tipos de pesquisa, ainda é possível utilizar o filtro **\[ F12 ] - Exibir as cotações que já se tornaram pedidos de compra**. Nesse caso, de acordo com o tipo de pesquisa selecionado, serão exibidos no grid os registros de cotação que já se tornaram pedidos de compra, ou seja, já foram enviados e foi obtido retorno da distribuidora.

Nessa seção, ainda é possível realizar o envio da cotação, através da tecla F4, que será o mesmo processo explicado em #enviar-cotacao.

Para fazer o lançamento, é necessário ter o XML da nota salvo no computador (normalmente o fornecedor envia por e-mail, se não for o caso, é possível solicitar o envio ao fornecedor ou baixar o arquivo pelo site da Sefaz). Após ter o XML salvo, basta clicar no botão <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FfP8WowR03SHw09EuAR0J%2FBotao%20NFe.png?alt=media&#x26;token=73737338-ca0e-4b8f-8f4d-4bda2bb51202" alt="" data-size="line">, e será aberta a tela para localização do mesmo. Selecione o arquivo e o sistema irá importar os dados e ficará disponível para fazer o processo de lançamentos de notas.

<mark style="color:red;">**Observação:**</mark> O sistema vai importar apenas arquivos **.XML**.

#### **Entrada via chave de acesso** <a href="#entrada-via-chave-de-acesso" id="entrada-via-chave-de-acesso"></a>

Esse tipo de entrada, é outra maneira indicada por ter um processo simples e o risco de erros ser praticamente nulo. Para isso, é necessário ter em mãos a chave de acesso da nota fiscal e o certificado digital da loja deve estar instalado no computador ou importado no sistema (para utilização do DMaster NFC-e). Após isso, basta digitar ou ler o código através de um leitor no **Campo Chave da NF-e** e clicar no botão <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fyczb5R3q2zWDK0zUTJsE%2FBotao%20NFe.png?alt=media&#x26;token=0cadc7d1-b7a9-4109-b82b-5e346fcff1d2" alt="" data-size="line">.

<mark style="color:red;">**Observações:**</mark>

* Caso o certificado digital esteja importado no sistema, ele irá fazer uma consulta e retornar os dados da nota.
* Caso o certificado digital esteja apenas instalado no computador, ele irá pedir para selecionar e confirmar o certificado desejado.

Após efetuar um dos processos a cima, o sistema irá importar os dados e ficará disponível para fazer o processo de lançamentos de notas.

### **Entrada manual** <a href="#entrada-manual" id="entrada-manual"></a>

Nesse tipo de entrada, é necessário informar todos os campos manualmente, sendo um processo muito trabalhoso, demorado e com alto risco de preenchimento incorreto. Por isso, sempre aconselhamos a entrada via XML ou por Chave.

Caso a opção escolhida seja pela entrada manual, o primeiro passo é preencher todos os campos com as informações fiscais da nota.

<mark style="color:red;">**Observação:**</mark> caso o fornecedor não esteja cadastrado, é necessário cadastrá-lo em [Fornecedores](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/v/documentacao-interna/documentacao-tecnica/farma-shop/cadastros#fornecedores). Todas as informações devem estar idênticas à nota física.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FVo6hcMYSIbZWryYxhgOz%2Fcabecalho.png?alt=media&#x26;token=bf620669-912f-4702-97ec-a3eedc98f9da" alt="" width="563"><figcaption></figcaption></figure>

Após preenchimento das informações fiscais da nota, é necessário inserir as informações dos produtos.

<mark style="color:red;">**Observação:**</mark> o lançamento dos produtos é individual.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FnubUNlIUr1v8NapiYee7%2FInformacoes%20produto.png?alt=media&#x26;token=547bc90a-c4e0-4984-89dd-603e78608ad4" alt="" width="536"><figcaption></figcaption></figure>

**Código Prod.:** campo destinado à identificação do produto. A pesquisa pode ser realizada por descrição, código interno ou código de barras. Após localizado o produto, todas as informações da linha serão preenchidas automaticamente de acordo com o cadastro do mesmo. Caso o produto ainda não esteja cadastrado, será necessário efetuar o cadastro do mesmo em [Cadastro de produtos](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/v/documentacao-interna/documentacao-tecnica/sngpc/integracao-farma-e-sngpc/utilizacao/cadastro-de-produtos).

**CFOP:** campo utilizado para inserir o CFOP do produto, código que identifica a natureza de circulação da mercadoria.

**CST:** identifica a origem da mercadoria e a forma de tributação que deverá incidir sobre a mesma de acordo com o Regime Tributário.

**Qt x Emb.:** campo preenchido automaticamente, de acordo com informação de quantidade de unidades da embalagem do cadastro do produto.

**Preço Custo:** campo destinado ao novo preço de custo do produto.

**Quant. NF:** campo utilizado para inserir a quantidade total do produto que está na nota fiscal.

**($) ST:** campo utilizado para informar o valor da Substituição Tributária em reais.

**(%) IPI:** campo utilizado para informar o valor do IPI em porcentagem.

**Total Item:** campo destinado ao valor total deste item (levando em conta o preço de custo e a quantidade de unidades informadas).

**Lote:** campo utilizado para inserir o lote do produto.

**Validade:** campo utilizado para informar a validade do produto.

**Ok:** após preenchimento de todos os dados, basta clicar em "Ok" e o produto será lançado no grid. Com isso, é necessário efetuar o mesmo processo para o próximo produto.

<mark style="color:red;">**Observação:**</mark> é necessário fazer este procedimento para todos os produtos presentes na nota.

Após o lançamento de todos os produtos, os mesmos serão mostrados no Grid. Com isso, deverão ser feito o processo de Lançamento.

#### Lançamentos de notas <a href="#lancamentos-de-notas" id="lancamentos-de-notas"></a>

O lançamento da nota será realizado após puxar a nota a partir de entrada citada a cima.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FDDx3VbF7WlwQUfrZCrLi%2FProdutos.png?alt=media&#x26;token=57387d7d-9aa0-4abd-a494-d2ea142fc27a" alt="" width="563"><figcaption></figcaption></figure>

Ao carregar a nota, os produtos são identificados com status, onde pode ser necessária alguma ação. Os status são:

<mark style="background-color:red;">**Produto não cadastrado**</mark>

Este alerta significa que o cadastro referente produto não foi encontrado (a base de pesquisa é o código de barras), é necessário associá-lo a um produto já existente no sistema ou efetuar um novo cadastro.

Para isso, basta utilizar a tecla **F2 – Localizar/Cadastrar Produto**.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FmngAp7JbUjO02D7vZcmf%2FLocalizar%20produto.png?alt=media&#x26;token=ea477709-2900-42ef-857a-a915678abfaf" alt="" width="563"><figcaption></figcaption></figure>

Ao utilizar a tecla F2, será aberta a tela para cadastrar, localizar ou selecionar o produto.

<mark style="color:red;">**Observação:**</mark> sempre utilizar a busca geral através da tecla F5.

Para acessar o cadastro do produto, é utilizada a **tecla** **F3** , onde será aberta a tela de cadastro de produto para cadastra-lo, nesse caso, o sistema irá puxar dados da nota fiscal para auxiliar o cadastro.\
Caso o código de barras já esteja cadastrado no sistema, é possível pesquisar o produto, utilizando a **tecla F2**. Após pesquisar o produto, utilizar a **tecla F1**, para selecionar o produto e associar ao produto que está chegando na nota.\
Além dos atalhos F1 e F2, é possível utilizar os botões <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FjjQvXSD9JcqNHMX5FZPH%2FSelecionar%20produto.png?alt=media&#x26;token=7c3a4289-918d-4d0a-87f5-1cf7b05a4a81" alt="" data-size="line"> e <img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FJgwNPYudt5wY3eMQE60q%2FPesquisar%20produto.png?alt=media&#x26;token=52ff6455-1a8a-400c-8fe9-fcb80bba947b" alt="" data-size="line">.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FIF9yvt31Dx3NXbCRbQgm%2FCadastrar%20produto.png?alt=media&#x26;token=09df6e34-558d-4293-9fed-dba24081e083" alt=""><figcaption></figcaption></figure>

<mark style="background-color:blue;">**Produto em promoção**</mark>

Este alerta significa que o cadastro referente ao produto foi encontrado, porém, o mesmo está com preço promocional configurado. Com isso, temos a opção de manter o preço promocional ou retirá-lo utilizando a tecla **F2 – Confirmar/Retirar Produto em Promoção**.

<mark style="color:red;">**Observação:**</mark> mesmo mantendo a promoção, os preços de custo e venda serão atualizados.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F14NwrutVJZXYkbCA9UHQ%2FProduto%20promocao.png?alt=media&#x26;token=d3bfe022-3acd-49e2-8815-a175fe3f3013" alt="" width="563"><figcaption></figcaption></figure>

Após utilizar a tecla F2 será aberta a tela de confirmação, basta clicar em Sim, para manter a promoção ou Não para removê-la. Uma nova tela de confirmação irá abrir, para manter ou retirar os demais produtos que estão em promoção.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FST1KvoIOcDn3P0QLieYA%2Fmensagens%20promocao.png?alt=media&#x26;token=07b32665-4a66-4e54-ae9a-31c1fe7f5bbb" alt=""><figcaption></figcaption></figure>

<mark style="background-color:orange;">**Divergência de preço**</mark>

Este alerta significa que o cadastro referente ao produto foi encontrado, porém existe divergência de preço de custo entre a nota fiscal e o custo atual do produto.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FYDgJD2b6AdyyzP02Ql9k%2FDivergencia%20de%20preco.png?alt=media&#x26;token=3c342752-2d6f-4c24-ae5c-6e850a5fba66" alt="" width="563"><figcaption></figcaption></figure>

Com isso temos duas opções:

* Fazer as alterações necessárias através da tecla **F2 – Alterar Preço de Venda - Custo Atual.**

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FJ7A2qtumLtyki7YsLEFd%2FAlteracao%20preco.png?alt=media&#x26;token=dc59f528-42a6-4639-9203-9e66a669b307" alt=""><figcaption></figcaption></figure>

Ao utilizar a tecla **F2 – Alterar Preço de Venda - Custo Atual**, será aberta a tela de Alteração de Preços, vai ser possível visualizar as informações atuais do cadastro do produto, abaixo vai ter as informações para ajuste, onde é possível alterar o preço de venda através do campo **Preço sugerido**, ou inserir a porcentagem de **Markup** e **Lucro**, e o preço de venda será alterado automaticamente. Após alterações, clicar em **Gravar**.

<mark style="color:red;">**Observação:**</mark> o preço de custo é buscado na nota fiscal, por tanto é bloqueado.

* Não fazer nenhuma alteração através da tecla **F3 – Ignorar**.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2Fzm4eDeRVLInqmdkZZFSp%2FIgnorar%20divergencia%20de%20preco.png?alt=media&#x26;token=c78a5bfb-02dd-4712-a1e9-060210307033" alt=""><figcaption></figcaption></figure>

Ao utilizar a tecla **F3 – Ignorar**, será aberta a tela de confirmação de variação de custo. É preciso ter atenção, pois nesta tela, é perguntado se deseja ignorar a variação de todos os produtos com alerta.

Para ignorar a variação de todos os itens, clicar em **Sim**.

Para ignorar a variação apenas do item selecionado, clicar em **Não**.

Para desistir do processo, clicar em **Cancelar**.

Após efetuar um dos processos, o produto irá ficar com o status Verificado ![](https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FywzaLMdb2qLuvZfgcuBc%2FStatus%20verificado.png?alt=media\&token=7d0747e3-8827-44b6-9387-677382d691f7).

<mark style="background-color:green;">**Produto verificado**</mark>

Este alerta significa que o cadastro referente ao produto foi encontrado e está verificado, seja por estar cadastrado e não ter alteração de preço, ou por ter sido cadastrado e/ou ter o preço ajustado.

<mark style="color:red;">**Observação:**</mark> Só é possível gravar a nota, quando todos os produtos estiverem com status verificado ![](https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FskHNY3ZYGA578z1hnEo8%2FStatus%20verificado.png?alt=media\&token=0c07e643-4b18-4fcc-b39b-20cfa9096cb7).

<mark style="background-color:yellow;">**Produto fracionado**</mark>

O produto fracionado é um produto que tem o estoque vendido em fração, como comprar caixas de remédio e vender as cartelas.

Dentro da entrada de notas, no campo **Qt Estq** deverá ser adicionada a quantidade total de frações (cartelas, comprimidos, unidade etc, no caso o tipo produto que será vendido). Dessa maneira o sistema irá lançar o estoque correto e ajustar o preço de custo.

<figure><img src="https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2F7O7CZ2MBqWZXQiSXTtY9%2FFracionado.png?alt=media&#x26;token=29d75334-936d-4aba-94d0-ae910c22d197" alt=""><figcaption></figcaption></figure>

**Observação:** só é possível fazer esse processo caso o produto já esteja cadastrado como fracionado, processo efetuado em **`Fracionamento`**

<mark style="background-color:purple;">**Material de uso/consumo e Ativo imobilizado**</mark>

<mark style="color:red;">**Observações:**</mark>

* Para produtos assinalados como uso/consumo ou ativo imobilizado, ao gravar a nota, o estoque e preços de custo e venda não são alterados. Além disso, é gerado um log o relatório de registro de eventos com esta informação **NÃO HOUVE ALTERAÇÃO DE ESTOQUE.**
* Esta opção só é exibida quando o produto está com status ![](https://981618378-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FaHl55K4HG6CuWy9zUjR9%2Fuploads%2FfPWHTdm9SdnwIdi1mJg4%2FStatus%20verificado.png?alt=media\&token=02c99821-a021-49d6-a2aa-b569bda9c220).

**F9 – Material de Uso ou Consumo:** opção utilizada para produtos de consumo próprio (ou da loja, ou pessoal), por exemplo, tinta para impressora, caneta, papel, produto de limpeza.

<mark style="background-color:yellow;">**Bonificação**</mark>

As informações de fornecedor, total dos produtos e produtos da nota serão carregados.\ <mark style="color:red;">**Observação:**</mark> a devolução realizada nesse módulo gera um registro de devolução no módulo de NFe, onde é possível gerar uma nota fiscal de devolução.
