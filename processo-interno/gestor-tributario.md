# Gestor Tributário

### Comercial

O setor comercial é responsável por negociar com o cliente todo o processo de contratação desse serviço.\
Caso o cliente abra um chamado solicitando informações para contratação do módulo, este deve ser enviado para o setor comercial realizar as tratativas.\
Após todo o trabalho do comercial ser finalizado, um chamado é encaminhado para o **setor operacional** com o título **"Inst. de Utilitários "** para que sigam com a configuração e treinamento do módulo.

### Setor Operacional

Ao receber o chamado, o consultor deverá realizar a configuração e treinamento do módulo, seguindo os passos que serão explicados abaixo.

#### Configuração

Realizar a configuração do módulo utilizando o aplicativo DMConfigurações, habilitando o Log padrão com o caminho do **Servidor**, de acordo com a documentação [https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/documentacao-tecnica/farma-shop/utilitarios](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/documentacao-tecnica/farma-shop/utilitarios "mention").

#### Treinamento

Sempre acessar o módulo com o usuário do cliente em questão e explicar ao mesmo as funcionalidades do módulo de acordo com a documentação de treinamento indicada abaixo [https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/processos-internos/treinamentos/conteudos/gestor-tributario](https://dmaster.gitbook.io/base-de-conhecimento-dmaster/documentacao-interna/processos-internos/treinamentos/conteudos/gestor-tributario "mention").

Ao final do treinamento, orientar o cliente a enviar a base completa dos produtos, ou seja, ativos e inativos.

{% hint style="info" %}
Por padrão, orientamos que o envio dos produtos seja realizado através do **Terminal Administrador da loja,** pois é um processo demorado e pesado. Mas caso o cliente necessite, o processo pode ser realizado em qualquer terminal da loja.

Deve ser informado ao cliente que o processo de envio dos produtos para o iMendes é demorado, e enquanto o processo é realizado, o terminal em questão não poderá ser utilizado.
{% endhint %}

#### Procedimentos pós treinamento <a href="#procedimentos-pos-treinamentos" id="procedimentos-pos-treinamentos"></a>

Após finalizar todo o treinamento e iniciar o envio dos produtos, orientar o cliente que fique atento ao processo e caso ocorra algum problema, ele abra um novo chamado, para que possamos auxiliá-lo.

Agendar o chamado para o **o seu nome**, para o **próximo dia útil** às **09:00hrs**. Caso o cliente ainda não tenha aberto um novo chamado, este poderá ser finalizado normalmente, sem necessidade de novo contato com o cliente.\
Se o cliente abrir um novo chamado indicando que houve problema no envio dos produtos, este deve ser finalizado com o título **"Solicit. Duplicada"** e o atendimento deve ser realizado pelo chamado que havia sido agendado anteriormente.
